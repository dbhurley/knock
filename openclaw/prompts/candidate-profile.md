# Candidate Profile Presentation Template

Use this template when presenting a candidate to a client school via Telegram. Format for readability in Telegram's message display.

---

## Template

```
--- CANDIDATE PROFILE ---

**{full_name}**{suffix_if_any}
{current_title}
{current_organization} | {city}, {state}

**Match Score**: {score}/100 {score_emoji}

---

**Current Role**
{current_title} at {current_organization}
Since {current_position_start_year} ({years_in_role} years)

**Career Stage**: {career_stage}

---

**Experience Highlights**
- {experience_1_title} at {experience_1_org} ({years})
- {experience_2_title} at {experience_2_org} ({years})
- {experience_3_title} at {experience_3_org} ({years})

**School Types**: {school_type_experience_list}
**Enrollment Range**: {enrollment_range}

---

**Education**
- {degree_1} -- {institution_1} ({year})
- {degree_2} -- {institution_2} ({year})

---

**Specializations**
{specialization_tags}

**Why This Candidate**
{match_reasoning}

---

**Availability**: {availability_note}
**Relocation**: {relocation_status}
**Compensation**: {compensation_bracket}

---

Knock Ref: {person_id_short}
```

---

## Field Definitions

| Field | Source | Notes |
|---|---|---|
| `full_name` | `people.full_name` | Include prefix (Dr., Rev.) if present |
| `suffix_if_any` | `people.suffix` | ", Ed.D." or ", Ph.D." -- append after name |
| `current_title` | `people.current_title` | |
| `current_organization` | `people.current_organization` | |
| `city`, `state` | `people.city`, `people.state` | |
| `score` | Computed by match_score skill | 0-100 integer |
| `score_emoji` | Derived from score | 90+: top candidate, 75-89: strong, 60-74: solid, below 60: developing |
| `career_stage` | `people.career_stage` | Capitalize: "Senior", "Mid-Career", etc. |
| `experience_*` | `person_experience` table | Top 3 most relevant roles |
| `school_type_experience_list` | `people.school_type_experience` | Comma-separated: "K-12, Boarding, Coed" |
| `enrollment_range` | `people.enrollment_experience_range` | e.g., "400-1,200 students" |
| `degree_*` | `person_education` table | Highest degrees first |
| `specialization_tags` | `people.specializations` | Hashtag style: #fundraising #DEI #boarding |
| `match_reasoning` | Generated by match_score | 2-3 sentences explaining why this candidate fits |
| `availability_note` | Derived from interactions/notes | "Available July 2026" or "Would consider for the right fit" |
| `relocation_status` | `people.willing_to_relocate` + `preferred_regions` | "Open to relocation" or "Prefers Northeast" |
| `compensation_bracket` | `people.compensation_expectation` | General range, never exact current salary |
| `person_id_short` | `people.id` | First 8 characters of UUID for reference |

---

## Score Emoji Guide

- 90-100: "Exceptional match"
- 80-89: "Strong match"
- 70-79: "Good match"
- 60-69: "Moderate match"
- Below 60: "Partial match -- review reasoning"

---

## Presentation Guidelines

1. **Never include** the candidate's personal email, phone number, or exact current salary in the profile presentation. These are shared only after the candidate has given consent to proceed.

2. **Match reasoning** should reference specific criteria from the search, not generic statements. Example: "12 years of boarding school leadership aligns with Greenfield's need for a Head familiar with residential life. Previous capital campaign at St. Mark's ($45M) matches the building program priority."

3. **When presenting multiple candidates**, show them in descending score order. Present 3-5 candidates at a time to avoid information overload.

4. **After presenting**, ask: "Would you like to learn more about any of these candidates, or shall I adjust the search criteria?"

5. **For shortlisted candidates** (those the client wants to advance), offer to prepare a detailed dossier with full career history, reference contacts, and assessment notes.
